이 문서는 모든 서비스(이커머스, SaaS, 소셜, 유틸리티 등) 기획 및 텍스트 생성 시 아래의 9가지 핵심 원칙을 엄격하게 준수하여 사용자의 인지 부하를 최소화하고 전환율을 극대화하기 위해 작성되었다.

---

## 1. 인지 부하 최적화 (Optimize Cognitive Load)

**Objective:** 사용자가 고민 없이 다음 행동을 결정할 수 있도록 퍼널(Funnel)을 설계한다.

* **Actionable:**
* 복잡한 판단이 필요한 질문을 배제하고, 직관적인 "네/아니오" 또는 단일 선택형 질문으로 재구성한다. (예: "어떤 방식으로 인증하시겠습니까?" -> "신분증을 가지고 계신가요?")
* 이탈률이 높은 병목 구간을 식별하여, 입력 단계를 쪼개거나 생략 가능한 단계를 뒤로 미룬다.


* **Constraint:** 사용자가 한 화면에서 3초 이상 고민하게 만드는 복잡한 선택지 제공 금지.

## 2. 직관적이고 긍정적인 UX 라이팅 (Conversational & Positive UX Writing)

**Objective:** 기계적인 시스템 언어를 배제하고, 사용자 친화적인 대화형 텍스트를 작성한다.

* **Actionable:**
* **일관된 대화형 톤앤매너:** 서비스 성격에 맞는 자연스러운 일상어를 사용하며, 전문 용어나 시스템 에러 코드를 일반인의 언어로 번역한다.
* **긍정 프레이징(Positive Phrasing):** "불가능합니다", "오류" 등의 부정어 대신, "이렇게 하면 해결할 수 있어요" 형태의 대안 제시형 문장으로 작성한다.
* **경어체 최적화:** 과도한 존칭이나 불필요한 서술어를 축약하여 텍스트의 스캐닝(Scanning) 효율을 높인다. (예: "등록을 진행하시겠습니까?" -> "등록할까요?")



## 3. 단일 목적 화면 설계 (One Thing per Page)

**Objective:** 화면의 정보량을 통제하여 사용자의 주의 분산을 막는다.

* **Actionable:**
* **1 Page = 1 Primary Action:** 한 화면에는 반드시 사용자가 완수해야 할 단 하나의 핵심 행동(Primary CTA)만 강조한다.
* **정보의 청킹(Chunking):** 시리얼 넘버, 주소, 연락처 등 긴 데이터는 시각적 덩어리로 분할(Formatting)하고 여백을 극대화한다.



## 4. 목적 지향적 시각 요소 및 트랜지션 (Functional Visuals & Transitions)

**Objective:** 심미성보다 '인지 보조'를 목적으로 그래픽과 애니메이션을 사용한다.

* **Actionable:**
* 상황을 설명하는 명확한 핵심 메타포(그래픽) 하나만 배치하며, 의미 없는 장식은 배제한다.
* 상태 변화나 작업 완료 시 부드러운 애니메이션을 통해 시스템 상태를 시각적으로 피드백한다.


* **Constraint:** 사용자에게 죄책감이나 불안감을 유발하여 이탈을 막는 '감정적 다크 패턴(Guilt-tripping)' 절대 사용 금지.

## 5. 범용적 접근성 (Universal Design & Accessibility)

**Objective:** 기기 환경, 신체적 조건에 구애받지 않는 사용성을 보장한다.

* **Actionable:**
* **반응형 텍스트 대응:** 사용자가 OS 설정에서 글씨 크기를 최대로 키워도 UI가 깨지지 않도록 설계한다. (필요시 리스트 뷰를 Bottom Sheet 등 다른 UI 컴포넌트로 전환하여 가독성 확보)
* **스크린 리더 최적화(A11y):** 시각 요소에 명확한 대체 텍스트(Alt Text)를 제공하고, 논리적인 포커스 이동 순서를 설계한다.



## 6. 인체공학적 인터랙션 및 피드백 (Ergonomic Interaction & Feedback)

**Objective:** 물리적인 터치 환경과 감각적 피드백을 설계한다.

* **Actionable:**
* **엄지 영역(Thumb-zone) 최적화:** 주요 CTA 버튼은 대화면 기기에서도 한 손으로 닿기 쉬운 화면 하단에 고정 및 확장 배치한다.
* **다중 감각 피드백:** 중요한 행동(결제 완료, 설정 변경 등) 시각적 변화와 함께 기기의 햅틱(진동) 피드백을 트리거하여 작업 완료를 명확히 인지시킨다.



## 7. 체감 성능 극대화 (Perceived Speed Optimization)

**Objective:** 물리적 로딩 시간을 줄일 수 없다면 심리적 대기 시간을 줄인다.

* **Actionable:**
* **스켈레톤(Skeleton) UI 적용:** 정적인 빈 화면이나 스피너(Spinner) 대신, 콘텐츠의 레이아웃 뼈대를 먼저 노출하여 시스템이 작동 중임을 알린다.
* **자동 포커스(Auto-advance):** 고정된 길이의 데이터(예: 인증번호 6자리) 입력이 완료되면 다음 폼(Form)이나 페이지로 터치 없이 자동 전환시킨다.



## 8. 맥락 기반 UI 컴포넌트 (Contextual UI Components)

**Objective:** 사용자가 결과를 예측할 수 있도록 돕고, 입력 과정의 허들을 낮춘다.

* **Actionable:**
* **예측 가능한 CTA (Action-oriented):** '확인/취소' 대신, 버튼 클릭 시 일어날 결과를 명확히 동사형으로 기재한다. (예: "장바구니에 담기", "구독 해지하기")
* **입력 최적화 키패드:** 입력해야 할 데이터 속성(숫자, 이메일, 전화번호 등)에 맞춰 가장 적절한 타입의 키보드(Numpad, Email pad 등)를 자동으로 호출한다.



## 9. 엣지 케이스에서의 공감 (Empathetic Edge Case Handling)

**Objective:** 에러나 빈 화면 등 예외 상황에서 사용자의 이탈을 방지하고 안심시킨다.

* **Actionable:**
* **행동 유도형 빈 화면 (Actionable Empty State):** 데이터가 없는 빈 화면(예: 빈 장바구니, 알림 없음)에는 상황 설명과 함께 서비스 사용을 시작할 수 있는 CTA나 튜토리얼을 배치한다.
* **안심시키는 에러 핸들링:** 시스템 오류 발생 시 원인을 사용자 탓으로 돌리지 않으며, 데이터(또는 자산)가 안전하다는 사실을 우선 고지한 뒤 명확한 해결책이나 우회 경로를 제공한다.
