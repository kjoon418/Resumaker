이 문서는, 서비스의 클라이언트 영역을 구현할 때, 사용자에게 에러 메시지를 어떻게 보여줄지를 명시하기 위해 작성되었다.

# 좋은 에러 메시지의 조건

## 최고의 에러는 발생하지 않는 것
목적: UX를 개선해서 불필요한 에러 메시지를 없앤다.

구체적인 개선 예시는 다음과 같다:
- 연락처 송금 화면에서 '내 연락처'를 누르면 "내 연락처로는 돈을 보낼 수 없어요"라는 에러 메시지를 보여주고 있다.
    -> 연락처 목록에서 '내 연락처'를 없애면, 사용자는 애시당초 해당 에러 메시지를 마주할 일이 없어진다.

## 적절한 컴포넌트를 사용한다
'적절하지 않은 컴포넌트 사용'의 예시는 다음과 같다:
- 충분한 설명이 필요한 상황인데 '토스트' 사용
    -> 사용자가 설명을 충분히 읽기 전에 컴포넌트가 사라진다.
- 위급한 상황이 아닌데 '빨간색' 등의 색상 사용
    -> 사용자가 에러의 심각도를 잘못 받아들이게 된다.

## 해결 방법을 알려준다
목적: 사용자가 에러를 해결하고 다음 단계로 넘어갈 수 있게 돕는다.

구체적인 개선 예시는 다음과 같다:
- "생체 인증에 실패했어요.
    -> "생체 인증 정보에 문제가 생겼어요. 같은 문제가 반복된다면 앱을 껐다 키거나 삭제하고 다시 설치해주세요."

## 사용자가 이해할 수 있는 언어를 사용한다
목적: 공급자 입장이 아니라 사용자 입장에서 보기 좋은 메시지로 만든다.

구체적인 개선 예시는 다음과 같다:
- "부적절한 신분증입니다."
    -> "기간이 만료된 신부증이에요. 체류 기간이 만료된 외국인 등록증이에요."
- "세션이 만료되었습니다."
    -> "시간이 지나 자동 로그아웃됐어요. 다시 로그인해주세요."

## 쉽게 해결할 수 있도록 돕는다
목적: 사용자가 최소한의 비용으로 에러를 해결할 수 있게 만든다.

구체적인 개선 예시는 다음과 같다:
- "신청에 실패했어요. 오늘 꼭 신청해야 한다면 고객센터(1234-5678)로 문의해주세요."
    -> 기존 메시지와 더불어, 고객센터에 문의하는 페이지로 이동할 수 있는 '문의하기' 버튼을 제시한다.

## 부정적인 감정을 최소화한다
목적: 사용자가 요청 실패 시 좌절감을 느끼지 않게 돕는다.

구체적인 개선 예시는 다음과 같다:
- "6자가 넘는 닉네임은 사용할 수 없습니다."
    -> "닉네임은 1~5글자 이내여야 해요."
- "이 서비스를 이용할 수 없는 연령입니다."
    -> "이 서비스는 만 19세 이상부터 이용할 수 있어요."
