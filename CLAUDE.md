# Project Context for AI Agent

이 문서는 이 프로젝트에서 작업하는 AI 에이전트를 위한 핵심 컨텍스트와 지침입니다. 코드를 작성하거나 수정하기 전에 반드시 이 문서의 규칙을 우선적으로 적용하세요.

## Project Overview (프로젝트 개요)
- **프로젝트명:** Resumaker
- **목적:** 사용자가 기록한 경험과 경력을 기반으로, 목적에 맞는 이력서/포트폴리오를 제공하는 서비스
- **주요 대상:** 내 경험과 경력을, 목표 기업에 최적화된 이력서/포트폴리오로 녹여내고 싶은 취업 준비생

## Tech Stack & Environment (기술 스택)
- **Mobile:** Kotlin & Jetpack Compose(MVP 단계에서는 모바일 구현 계획 없음)
- **Web**: Compose Multiplatform (Wasm/Canvas)
- **Backend/API:** Kotlin Spring Boot
- **Database:** 트레이드 오프를 고려해 선택(확정 후 현재 명세 수정 필요)

## 에이전트 행동 지침
- `docs/` 디렉터리 하위에 행동 지침 파일이 배치되어 있습니다.
- 코드를 작성하거나 수정할 때, 반드시 해당 지침을 기반으로 수행해야 합니다.
- 한 목적을 달성하기 위해 사용할 수 있는 도구가 여럿 있을 수 있습니다.
    - 이 때는, 각 방법의 트레이드 오프를 나열한 후, 현재 상황을 기반으로 근거를 들어 한 방법을 선택합니다.
    - 각 방법의 트레이드 오프를 나열했을 때 한 방법을 선택할 수 없다면, 사용자에게 선택지를 제시합니다.

## 버전 관리
- 커밋은 다음 조건을 충족하도록 나눈다.
    - 서비스가 온전히 동작해야 한다.
    - 1 ~ 2개의 문장으로 표현할 수 있어야 한다.
    - 작업 단위마다 하나의 커밋으로 모은다.
    - 작성한 프로덕션 코드에 대한 테스트를 작성해야 하며, 테스트가 통과되어야 한다.
- 커밋 메시지는 the AngularJS commit conventions를 기반으로 작성한다.
- 예시는 다음과 같다.
    - "feat: 회원 가입 구현"
    - "fix: 상품 구매 로직의 버그 해결"
    - "test: 주문 시나리오에 대한 E2E 테스트 작성"

## 도메인 이해
- `docs/Resumaker 도메인 이해.md` 파일을 통해, 도메인에 대한 명세를 확인할 수 있습니다.
- 모든 기능 추가, 성능 개선, 기술 도입은 도메인 관점에서 이루어져야 합니다.
- 구현 과정에서 도메인 명세 자체에 문제점이 있다고 판단될 경우, 도메인 명세 수정을 요청할 수 있습니다.

## 지침 및 명세서 개선
- 다음과 같은 상황에서 지침 및 명세서 개선을 요청할 수 있습니다.
    - 지침을 지키는 과정에서 코드 품질 저하가 우려되는 상황.
    - 도메인 명세서 자체에 모순, 문제점, 개선점이 존재하는 상황.
    - 사용자의 요구사항을 처리할 때 참고할 수 있는 지침이 존재하지 않는 상황
- 다음과 같은 상황에서는 지침 및 명세서를 자동으로 개선합니다.
    - 사용자와의 합의를 통해, 도메인 명세나 기술 스펙 자체가 달라진 상황.
- 기존 지침대로 수행했을 떄와 새로운 지침을 적용했을 때를 비교해 트레이드 오프를 명확히 제시해서 요청해야 합니다.

## 반문
- 다음과 같은 상황에서는 스스로 추측한 후 작업을 진행하는게 아니라, 사용자에게 본인이 이해한 내용이 맞는지 되묻습니다.
    - 사용자가 요구사항을 모호하게 제시하여, 어떤 작업을 수행해야 할지 확실히 알 수 없는 상황.
    - 요구사항대로 수행하면, 기존 지침이나 명세를 위반하게 되는 상황.
